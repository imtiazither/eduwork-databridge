# Security Policy Summary

Report vulnerabilities privately through GitHub private vulnerability reporting when enabled. Do not include real data or credentials in reports.

Security-relevant areas include secret exposure, cross-organization access, authorization bypass, unsafe files/archives, SSRF, injection, dependency compromise, unauthorized identity merges, export leakage, audit tampering, and retention-path escape.

See the repository `SECURITY.md` for the authoritative disclosure process and supported-version statement.
