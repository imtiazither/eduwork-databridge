# Governance

The maintainer owns release decisions while the project is in pre-1.0 development. Changes to public contracts, canonical entities, security boundaries, migrations, licensing, or claim language require an issue and Architecture Decision Record.

## Decision principles

1. Truthful maturity labels
2. Synthetic-only public data
3. Backward-compatible public contracts where possible
4. Reversible, attributable data decisions
5. Human review for ambiguity
6. Minimal necessary data
7. Tests and documentation in the same pull request as behavior changes
